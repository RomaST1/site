# SmartHome OOP

Система керування розумним будинком, створена на JavaScript із використанням модульної архітектури та Supabase як backend.

## 📌 Опис проєкту

SmartHome OOP — це веб-застосунок для керування пристроями розумного будинку. Система дозволяє:

- переглядати список пристроїв;
- додавати, редагувати та видаляти пристрої;
- керувати користувачами;
- виконувати авторизацію за ролями;
- переглядати статистику станів пристроїв;
- надсилати повідомлення через форму контактів.

Проєкт побудований із використанням модульного JavaScript та інтеграції з Supabase.

# 🚀 Основний функціонал

## 🔐 Авторизація

- вхід користувачів;
- збереження сесії через `localStorage`;
- розмежування доступу за ролями:
  - `admin`
  - `manager`
  - `user/client`

## 🏠 Керування пристроями

- додавання пристроїв;
- редагування параметрів;
- видалення пристроїв;
- пошук і фільтрація;
- пагінація;
- статистика по статусах.

## 👥 Керування користувачами

- створення користувачів;
- редагування;
- видалення;
- контроль ролей.

## 📩 Контактна форма

Повідомлення користувачів автоматично зберігаються у таблиці `messages` в Supabase.

# 🧠 Архітектура

## `api.js`

Робота з Supabase:

- `query()`
- `insert()`
- `update()`
- `remove()`

## `auth.js`

Логіка:

- авторизації;
- перевірки ролей;
- logout;
- меню користувача.

## `render.js`

Відповідає за:

- рендер таблиць;
- карток пристроїв;
- статистики;
- пагінації.

## `actions.js`

CRUD-операції:

- редагування;
- видалення;
- фільтрація.

## `utils.js`

Допоміжні функції:

- кольори статусів;
- утиліти інтерфейсу.

# 🗄️ Supabase

Проєкт використовує Supabase як backend.

Основні таблиці:

## `clients`

| Поле | Опис |
|---|---|
| id | ID пристрою |
| name | Назва |
| company | Кімната/локація |
| email | Статус/опис |
| phone | Серійний номер |
| status | Стан |

## `users`

| Поле | Опис |
|---|---|
| id | ID |
| fullName | Ім’я |
| username | Логін |
| password | Пароль |
| role | Роль |

## `messages`

| Поле | Опис |
|---|---|
| name | Ім’я |
| subject | Тема |
| message | Повідомлення |
| status | Статус |

## 1. Клонування репозиторію
